## 3 Core Principles for Vetting Technical Competence

Before reviewing proposals or signing retainer agreements, force prospective agencies or consultants to explain how they handle three core engineering realities:

### 1. The Princeton GEO Benchmark Test
Research from Princeton and Georgia Tech on Generative Engine Optimization (GEO) proved that increasing blog word count does not drive AI citations. **Structure does.**

*   **+34% Retrieval:** Achieved simply by adding hard, verified statistical data points to a page.
*   **+44% Retrieval:** Achieved by integrating named, verified expert quotations.
*   *Executive Takeaway:* If an agency's proposal centers on using AI tools to churn out dozens of long-form articles, they are generating noise that algorithms are explicitly built to ignore.

### 2. Local Entity Graph Seeding
AI models cross-examine your corporate claims against external data clusters. 

*   Take regional technology hubs like [AI Chiang Mai](https://aichiangmai.com) as a working example. Platforms like this build domain authority not through 2,000-word fluff pieces, but by seeding clean, interconnected data (developer profiles, physical coordinates, verified project nodes) directly to web scrapers.
*   *Executive Takeaway:* A qualified specialist knows how to connect your corporate identity to trusted external clusters so engines instantly verify who you are.

### 3. Edge Pipeline & Server Auditing
Your web architecture must serve clean data at peak speeds. A true Answer Engine Optimization (AEO) audit inspects raw server logs, validates nested JSON-LD schema, and deploys a root `llms.txt` file. 

*   *Executive Takeaway:* If your primary website relies on heavy, unoptimized client-side JavaScript, AI crawlers will time out and drop your connection before ever reading your content.

### 1. Server Log & LLM Crawler Management
Ask candidates to show you a live server log audit. Generative engines use specialized crawlers with strict rate limits:
*   `GPTBot` / `ChatGPT-User` (OpenAI)
*   `PerplexityBot` (Perplexity Search)
*   `Claude-Web` (Anthropic)

If an agency isn't tracking these bots inside server access logs, they are operating without critical operational data. They should also configure a root `llms.txt` file—a clean, Markdown directory that helps AI engines digest your product details without wading through heavy website code.

### 2. Entity Mapping & Nested JSON-LD Arrays
Basic plugins that output standard "BlogPosting" schema are no longer enough. Answer engines rely on explicit **entity graphs** to verify your business identity:

```json
{
  "@context": "[https://schema.org](https://schema.org)",
  "@graph": [
    {
      "@type": "Organization",
      "@id": "[https://yourdomain.com/#organization](https://yourdomain.com/#organization)",
      "name": "Your Brand Name",
      "url": "[https://yourdomain.com](https://yourdomain.com)",
      "sameAs": [
        "[https://www.wikidata.org/wiki/Q123456](https://www.wikidata.org/wiki/Q123456)",
        "[https://crunchbase.com/organization/yourbrand](https://crunchbase.com/organization/yourbrand)"
      ],
      "knowsAbout": [
        "[https://en.wikipedia.org/wiki/Search_engine_optimization](https://en.wikipedia.org/wiki/Search_engine_optimization)",
        "[https://en.wikipedia.org/wiki/Generative_engine_optimization](https://en.wikipedia.org/wiki/Generative_engine_optimization)"
      ]
    }
  ]
}
```

If an agency cannot draft or explain custom relational schema arrays, they cannot effectively secure citations across ChatGPT, Perplexity, and Gemini.

### 3. Content Architecture: Information Gain vs. Word Count
Traditional agencies bill by word count, delivering bloated 3,000-word articles. Language models calculate information density per token—if a page takes 1,000 words to answer a question, the model penalizes it in favor of a concise source.

When instructing an agency to optimize our content framework, evaluate their approach:
*   **Legacy Agency:** "We will add 1,000 words and target secondary long-tail keywords."
*   **GEO Specialist:** "We will restructure core pages into clean Answer Blocks, place a 60-word factual summary at the top, and convert narrative lists into structured data tables."

### 4. Audit Quality: Real Engineering vs. Automated Scans
When you hire a firm to conduct an AEO audit, review their deliverables carefully. If they hand you a 50-page automated PDF export from generic software, you are working with an account manager, not a systems engineer.

A technical audit must analyze:
*   **DOM Hydration:** Verifying content loads in the initial server response rather than relying on delayed JavaScript execution.
*   **Crawl Ratios:** Pruning low-value pages that dilute overall domain authority.
*   **Vector Testing:** Simulating how AI models retrieve citations from your domain versus market competitors.

## 5 Direct Questions to Ask Before Signing a Retainer

Use these direct questions during discovery calls to test an agency's technical depth:

1.  **"How do you track whether our domain is being cited inside ChatGPT or Perplexity search results?"**
    *   *Pass:* "We run automated testing scripts via direct API endpoints to track prompt citation frequency and monitor referral paths in server logs."
2.  **"How do you handle client-side JavaScript execution when auditing for AI crawlers?"**
    *   *Pass:* "We implement Server-Side Rendering (SSR) or static pre-rendering so clean HTML is delivered on the very first HTTP response."
3.  **"What is your process for connecting our brand node to recognized entity databases?"**
    *   *Pass:* "We seed a consistent entity footprint across verified platforms and explicitly connect those relationships inside custom `@graph` schema arrays."
4.  **"How do you restructure existing web pages that fail to pull AI citations?"**
    *   *Pass:* "We trim fluff, add concise summary blocks at the top, convert narrative points into semantic HTML tables, and attach microdata attributes."
5.  **"What hosting architecture do you recommend for maximum crawl speed?"**
    *   *Pass:* "We advocate for modern, decoupled edge architectures (like Astro or Next.js) deployed via Cloudflare or Vercel to guarantee sub-100ms response times globally."
